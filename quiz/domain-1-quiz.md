## Answers

| Q | Answer | Explanation |
|---|--------|-------------|
| 1 | **B** | SAP EA is built on TOGAF and tailors it — it does not replace it |
| 2 | **C** | Cloud migration = IT-driven engagement → IT-Centric Metro Map |
| 3 | **A** | No dedicated EA team, EA skills not yet formalised = Informal model |
| 4 | **C** | Signavio = process analysis, 1000+ metrics, 24-hour connection |
| 5 | **C** | Step 4 = LeanIX roadmaps handed to Cloud ALM, projects generated |
| 6 | **D** | The 5 blocks are: Methodology, Content, Tooling, **Practice**, Services. Governance is part of Practice. |
| 7 | **B** | Deliverables = Architecture Development. Not influencing decisions = Action & Impact. |
| 8 | **B** | Iterative — within phases, between phases, between cycles |
| 9 | **B** | Preliminary Phase = setup, ADM selection, tailoring |
| 10 | **B** | Loosely coupled divisions with different architectures = Separated |
| 11 | **C** | SAP EAF officially launched in 2007 |
| 12 | **C** | Artifacts are selected for the sake of stakeholders, not for architectural completeness |
| 13 | **B** | Initial = before mitigation. Residual = after mitigation actions applied. |
| 14 | **C** | "Linking applications and processes for better impact analysis" |
| 15 | **C** | ADRs must be created BEFORE decisions are taken. Architecture after decisions = too late. |
