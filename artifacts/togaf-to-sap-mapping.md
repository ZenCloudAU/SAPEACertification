# TOGAF to SAP EA Mapping

> For anyone coming from TOGAF or encountering TOGAF terms in the exam.

---

## Term Mapping

| TOGAF Term | SAP EA Term | Notes |
|-----------|-------------|-------|
| Driver | **Strategic Priority** | High-level forces shaping architecture |
| Objective | **Value Driver** | Measurable outcomes that deliver value |
| Gap | **Gap** | Same term — difference between baseline and target |
| ADM | **SAP EA Methodology** | SAP's tailored version of the ADM |
| Architecture Repository | **SAP LeanIX** | The tool that serves as the EA repository |
| Architecture Board | **EA Governance Body** | Oversees architectural decisions |
| BDAT (Business, Data, Application, Technology) | **Business, Data, Application, Technology** | Same four domains |
| TRM (Technical Reference Model) | **SAP TRM** | SAP's SAP-to-TOGAF TRM mapping |

## ADM Phase Mapping

| TOGAF ADM Phase | SAP EA Phase | Key Artifacts |
|----------------|-------------|---------------|
| Preliminary | Preliminary | Tailored ADM, Principles Catalog, Metro Map selection |
| Phase A: Architecture Vision | Architecture Vision | Stakeholder Map, Strategy Map, BMC, Solution Context/Concept, Statement of Architecture Work |
| Phase B: Business Architecture | Business Architecture | Org Map, Capability Map, Process Model, Business Footprint |
| Phase C: IS Architecture | Application & Data Architecture | Product Map, Solution Component Diagram, Process Flow, Software Distribution |
| Phase D: Technology Architecture | Technology Architecture | Environments & Location Diagram |
| Phase E: Opportunities & Solutions | Opportunities & Solutions | Roadmaps, Transition Architectures, WBS |
| Phase F: Migration Planning | Migration Planning | Implementation & Migration Plans |
| Phase G: Implementation Governance | Implementation Governance | Architecture Contract, Compliance Assessments |
| Phase H: Architecture Change Management | Architecture Change Management | Change Requests, Architecture Updates |
| Requirements Management | Requirements Management | Requirements Catalog (runs through all phases) |

## What SAP Added to TOGAF

SAP extended TOGAF with:
1. **User guidelines** — framework adaptation, engagement initiation, capability assessments
2. **Architecture method extensions** — worksheets per phase (inputs, steps, outputs)
3. **Case studies and examples** — real-world SAP architecture engagements
4. **Content metamodel** — architecture metamodel aligned with TOGAF terms
5. **Resource base extensions** — SAP business maps, SAP-TOGAF TRM, SAP product availability matrix, reference model content
6. **SAP Mapping Extensions** — SAP EAF terminology ↔ SAP terminology; SAP product ↔ TOGAF TRM; SAP method ↔ SAP method
7. **SAP Tooling Extensions** — enables EA tool mapping (LeanIX, Signavio, Cloud ALM)

---

## What Did NOT Change from TOGAF

- The **iterative nature** of the ADM — SAP EA is still iterative
- The **four architecture domains** — Business, Application, Data, Technology
- The **concept of Architecture Principles** — 4 components (Name, Statement, Rationale, Implications)
- The **Requirements Management** cross-cutting process
- The fundamental **as-is → to-be → gap analysis → roadmap** logic
- The importance of **stakeholder engagement**

---

## TOGAF Certification Recommendation

- TOGAF certification is **not mandatory** for the SAP EA certification
- It is **strongly recommended** as preparation — provides the foundational mental models
- If you have TOGAF, you have ~40% of the conceptual knowledge already
- SAP EA builds on TOGAF — knowing TOGAF first means you learn the SAP additions, not everything from scratch
